## Archivos importantes
-- "diseño tablas.md": Este archivo contiene todas las tablas de la base de datos, contiene tablas, columnas de las tablas y tipo de dato de cada columna, adicionalemente de una breve explicacion de cada tabla o bloque de tablas. Su ubicacion se encuentra en docs/diseño tablas.md
-- "Investigación Caso#3.md": Este archivo se encuentra la investagación que se realizo para crear el diseño de las tablas de la base de datos y informacion extra para conocer un poco sobre como funcionan este tipo de aplicaciones o paginas web de apuestas. Su ubicacion se encuentra en docs/Investigación Caso#3.md
-- "Investigacion flyway.md": Este archivo contiene toda la investigación que se realizo conforme a flyway junto a SQL server, contiene: que es y que hace, como se instala, como se configura, como es la estructura de carpetas, versionamiento, ejecucion de migraciones, rollback y hallazgos importantes. Su ubicacion se encuentra en docs/Investigacion flyway.md
-- "Patrones de diseño para usar explicados.md": Los patrones de diseños con su explicación los cuales se aplicaron en el diseño de las tablas de la base de datos y algunos que se podrian haber aplicado. Su ubicacion se encuentra en docs/Patrones de diseño para usar explicados.md
-- "Scripts Flyway.md": Explica que hace cada uno de los tres scripts de migracion Flyway (creacion del esquema, datos de catalogo y seeding), la estructura de carpetas y como reaplicar la base desde cero. Su ubicacion se encuentra en docs/Scripts Flyway.md. Los scripts en si estan en flyway/sql.
-- "Documentacion niveles de aislamiento.md": Archivo donde se explicara cuales son los problemas que pueden ocurrir en cada nivel, como identificarlo y como mitigarlos o solucionarlos en escenarios reales, los niveles que se hablan son "READ UNCOMMITTED", "READ COMMITTED", "REPEATABLE READ" y "SERIALIZABLE". La ubicacion de este archivo es docs/Documentacion niveles de aislamiento.md
-- "respuestaGemini.md": En este archivo se muestra algunas correciones que nos dio gemini por primera vez al consultarle sobre nuestra base de datos. Su ubicacion se encuentra en docs/historialGemini/respuestaGemini.md.
-- "respuestaGemini 2.0.md": Este archivo se muestra algunas correciones que nos dio gemini por segunda vez al consultarle sobre nuestra base de datos. Su ubicacion se encuentra en docs/historialGemini/respuestaGemini 2.0.md
-- "respuestaGemini 3.0.md": Este archivo se muestra algunas correciones que nos dio gemini por tercera vez al consultarle sobre nuestra base de datos. Su ubicacion se encuentra en docs/historialGemini/respuestaGemini 3.0.md 
-- "flyway.conf": Este archivo es el archivo maestro de flyway, es como un docker compose, este contiene URL de conexion JDBC para Microsoft SQL Server, tiene el usuario y contraseña, ubicacion de los scripts de migracion, esquema por defecto, la validacion de hashes al migrar para detectar cambios no autorizados en scripts ya aplicados, esto si lo queremos entonces esta en true, decide si permitir migraciones fuera de orden o no, en nuestro caso no queremos eso y elegir el tipo de codificacion de los scripts SQL como se usa en todo es UTF-8. La ubicacion de este archivo es flyway/conf/flyway.conf
-- ".gitkeep": Este archivo es para que git reconozca las carpetas de drivers y jars ubicadas en flyway/drivers y flyway/jars respectivament.s
-- "V1.0__Create_Schema.sql": Este archivo es el sql para crear las tablas con el formato de flyway. La ubicacion del archivo es flyway/sql/V1.0__Create_Schema.sql
-- "V1.1__Insert_Catalog_Data.sql": Este archivo es el sql con la inserción de los datos a la base de datos, los datos que se van a ingresar son: PAISES, ESTADOS, CIUDADES, ROLES, PERMISOS, PERMISOS POR ROL(Admin, moderador y jugador), MONEDAS, TIPOS DE CONTACTO, ESTADOS DE PROPOSICION, TIPOS DE RESULTADO DE PROPOSICION, ESTADOS DE PREDICCION, PLATAFORMAS SOCIALES, TIPOS DE RECURSO SOCIAL, PROVEEDORES DE IA, MODELOS DE IA, METODOS DE PAGO, METODOS DE PAGO POR PAIS, TIPOS DE MOVIMIENTO, TIPOS DE RESULTADO DE PAGO, SEVERIDADES DE ERROR, TIPOS DE EVENTO, OBJETOS DE DATOS, TIPOS DE OBJETO DE REFERENCIA, TIPOS DE OBJETO FUENTE, TIPOS DE PROCESO IA, TIPOS DE CONTENIDO IA, TIPOS DE FUENTE IA, TIPOS DE RESULTADO IA, TIPOS DE URL, CONFIGURACIONES DEL SISTEMA . La ubicacion del archivo es flyway/sql/V1.1__Insert_Catalog_Data.sql
-- "V2.0__Seeding.sql": Este archivo es una inseción de los datos de 1 usuario Sistema + 1000 jugadores y 5000 proposiciones. La ubicacion del archivo es flyway/sql/V2.0__Seeding.sql
-- "Concurrencia y Transacciones.md": Documenta los conceptos de SP anidados, niveles de aislamiento y deadlocks: que problemas provoca cada nivel, como identificarlos y como mitigarlos. Su ubicacion se encuentra en flyway/concurrencia/Concurrencia y Transacciones.md. Los scripts de demo estan en la misma carpeta (01_SP_Anidados.sql, 02_Aislamiento.sql, 03_Deadlocks.sql).
